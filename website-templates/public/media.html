Coaching

Learn to direct AI like an operator

You do not need to become a developer. You need habits: clear asks, review steps, and knowing which tool fits the job. I teach what I use daily to ship demos and client work.

  • Prompt like a product owner. One painful workflow, one success metric — not “automate everything.” How to write asks the model can actually execute.
  • Agents vs automations. When the model should read and interpret (documents, drafts, triage) vs when fixed rules should run every time (webhooks, booking, integrations).
  • Review before trust. Human-in-the-loop patterns: tables you edit before export, checklists before send, and what never goes fully autonomous on day one.
  • Cursor and agent workflows. How non-devs can still steer: commands, scoped tasks, subagents for big maps, and stopping before the stack becomes magic theater.
  • Document workflows for business. PDFs, invoices, receipts — extraction, validation, and Excel handoff without black-box “AI said so.”
  • Ship something clickable. End sessions with a demo URL or screen recording habit — proof beats slides for employers and clients.

Formats: 1:1 sessions, small cohorts when scheduled, or a walkthrough tied to your stack. See the live Documentation Agent on Agents first if you want context.
